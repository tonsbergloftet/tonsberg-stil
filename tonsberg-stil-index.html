Struktur

Scroll-snap presentasjon

Alle presentasjoner bruker scroll-snap. Én slide = ett poeng. Alltid en Neste-knapp nederst.

<!-- Grunnstruktur -->
<div class="slides" id="slides">
  <section class="slide" data-index="0">
    <div class="slide-inner">
      <!-- innhold -->
    </div>
    <button class="next-btn" onclick="goNext(0)">↓ Neste</button>
  </section>
</div>
/* Nødvendig CSS */
html, body { height: 100%; overflow: hidden; }

.slides {
  height: 100vh;
  overflow-y: scroll;
  scroll-snap-type: y mandatory;
  scroll-behavior: smooth;
}

.slide {
  scroll-snap-align: start;
  min-height: 100vh;
  position: relative;
  display: flex;
  flex-direction: column;
  background: var(--bg);
  overflow: hidden;
}

.slide-inner {
  flex: 1;
  display: flex;
  flex-direction: column;
  justify-content: center;
  max-width: 820px;
  margin: 0 auto;
  padding: 60px 36px 100px;
  width: 100%;
  position: relative;
  z-index: 5;
}

Callout

Sage-grønn venstrelinje. Brukes til sitater, nøkkelpoeng og oppsummeringer.

Det handler ikke om mer innsats. Det handler om å koble innsatsen bedre — slik at barn, unge og familier slipper å være prosjektleder for eget liv.

Prinsipper

Innhold

PrinsippDetalj
Én overskrift per slideBærer ett poeng alene. Overskriften er en påstand, ikke et temaord.
Alltid en visuell komponentAldri bare tekst og overskrift. Bruk kort, tabell, bevegelsesmåler eller dashboard.
Callout til nøkkelsetningenDen setningen deltakerne skal huske går i callout.
Spørsmålsblokk avslutterSiste slide er alltid refleksjonsspørsmål, ikke konklusjon.
Norsk bokmålKort og direkte språk. Brødtekst maks 3 setninger utenfor komponenter.

Visuelt

PrinsippDetalj
Square Peg som innslagAldri som primærfont. Kun til overskriftsinnslag, knapper og numre.
Trafikklys konsekventRust = problem, amber = mellom, green = positiv. Aldri andre farger for status.
Navy er autoritærBrukes til alt strukturelt — topbar, bølge, overskrifter.
Sage er varmBrukes til bølger, callout og sekundære elementer.
Illustrasjoner fremfor emojiEmoji kun i kortkomponenter. Aldri fritt i layouten.
